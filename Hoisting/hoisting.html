JavaScript Hoisting

Hoisting is JavaScript's default behavior of moving declarations to the top before executing the code.

1. What is Hoisting?

Hoisting is JavaScript's behavior of moving variable and function declarations to the top of their scope before code execution. 

console.log(a);

var a = 10;

2. Memory Creation Phase

Before executing the program, JavaScript allocates memory for variables and functions. 

var → undefined

let / const → TDZ

Function Declaration → Entire function stored

3. Execution Phase

JavaScript executes the code line by line and assigns values to variables. 

var a = 10;

4. Temporal Dead Zone (TDZ)

Variables declared using let and const cannot be accessed before their declaration. 

console.log(a);

let a = 10;

5. Function Declaration Hoisting

Function declarations are completely hoisted and can be called before they are declared. 

show();

function show(){

}

6. Function Expression Hoisting

Function expressions are not fully hoisted because only the variable declaration is hoisted. 

display();

var display = function(){

}
